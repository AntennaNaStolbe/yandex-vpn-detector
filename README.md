# Яндекс определяет VPN даже при split tunneling — разбор и решение

## 📌 Проблема

С середины апреля 2026 года многие пользователи столкнулись с проблемой:

Приложения Яндекса (Лавка, Еда, Почта, Погода и др.) начали показывать сообщение:

> «Похоже, у вас используется VPN»

и блокировать доступ.

При этом:
- VPN настроен **только для отдельных доменов и подсетей (split tunneling)**
- Сервисы Яндекса идут **напрямую через провайдера (без VPN)**

---

## ⚙️ Моя конфигурация

- Роутер: Keenetic  
- VPN настроен с маршрутизацией по доменам и подсетям  
- Домены Яндекса **не направляются через VPN**

Несмотря на это, приложения всё равно определяют VPN.

---

## 🔍 Как это проверялось

С помощью Fiddler был перехвачен трафик iPhone.

Сравнивались два сценария:
1. VPN включен (split tunneling)
2. VPN полностью отключен

В результате был найден ключевой запрос:

POST https://mobileproxy.passport.yandex.net/tmgrdfrend/checkvpn

---

## 📊 Поведение сервера

### ❌ Когда VPN определяется

Заголовок ответа:
X-Yandex-EU-Request: 1

Тело ответа содержит:
/vpndeny?v=fb

➡️ Приложение понимает, что используется VPN  
➡️ Показывается блокирующая плашка

---

### ✅ Когда трафик идет напрямую

Заголовок:
X-Yandex-EU-Request: 0

Тело ответа:
0

➡️ Проверка пройдена  
➡️ Приложение работает нормально

---

## ⚠️ В чем причина

Даже если домены Яндекса исключены из VPN:

Домен:
mobileproxy.passport.yandex.net

может:
- резолвиться через VPN DNS
- попадать под VPN из-за подсетей
- маршрутизироваться через VPN из-за правил роутера

➡️ Этого достаточно, чтобы Яндекс определил соединение как VPN

---

## ✅ Решение (на примере Keenetic)

### ✔️ Вариант 1 — правильный

1. Создать список доменов
2. Добавить:
   mobileproxy.passport.yandex.net
3. Назначить маршрут:
   → строго через провайдера (без VPN)

✔️ После этого приложения перестают ругаться

---

### ⚡ Вариант 2 — быстрый (костыль)

Заблокировать домен:

Через DNS-фильтр или hosts:
0.0.0.0 mobileproxy.passport.yandex.net

Поведение:
- если запрос падает с ошибкой — приложение пропускает пользователя

⚠️ Возможны побочные эффекты (например, при авторизации)

---

## 🧠 Итог

- Яндекс использует отдельный endpoint для проверки VPN  
- Split tunneling не гарантирует, что VPN не будет обнаружен  
- Достаточно одного «неправильного» домена

✔️ Решение занимает 1–2 минуты  
✔️ Не нужно постоянно включать/выключать VPN

---
